````markdown
## 1.1 创建订单

`POST /api/order/create`

**接口说明**：创建新订单

> **业务规则**：
> - 同一用户同一商品每日限购 5 件
> - 订单金额超过 10000 元需人工审核

**接口目录**：
- [1.1.1 请求信息](#111-请求信息)
- [1.1.2 请求体](#112-请求体)
- [1.1.3 响应结构](#113-响应结构)

### 1.1.1 请求信息

| 项目 | 值 |
|------|-----|
| 请求 Content-Type | `application/json` |
| 响应 Content-Type | `application/json` |

### 1.1.2 请求体

#### 根对象

| 字段名 | 类型 | 必填 | 说明 | 枚举值 |
|--------|------|------|------|--------|
| orderNo | String | 是 | 订单编号 | - |
| status | String | 是 | 订单状态 | 见 [OrderStatus 枚举](#orderstatus-枚举) |
| items | Array\<OrderItem\> | 是 | 订单明细 | 见 [OrderItem 对象](#orderitem-对象) |

#### OrderItem 对象

| 字段名 | 类型 | 必填 | 说明 | 枚举值 |
|--------|------|------|------|--------|
| productId | Long | 是 | 商品ID | - |
| quantity | Integer | 是 | 数量，范围: 1-999 | - |

**JSON 示例**：

```json
{
  "orderNo": "ORD20240115001",
  "status": "PENDING",
  "items": [
    {
      "productId": 1001,
      "quantity": 2
    }
  ]
}
```

### 1.1.3 响应结构

#### data 对象

见 [OrderRes 对象](#orderres-对象)

**JSON 示例**：

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "id": 10001,
    "orderNo": "ORD20240115001",
    "status": "PENDING"
  }
}
```

## 公共对象定义

### OrderRes 对象

| 字段名 | 类型 | 说明 | 枚举值 |
|--------|------|------|--------|
| id | Long | 订单ID | - |
| orderNo | String | 订单编号 | - |
| status | String | 订单状态 | 见 [OrderStatus 枚举](#orderstatus-枚举) |

---

## 枚举定义汇总

### OrderStatus 枚举

| 枚举值 | 描述 |
|--------|------|
| PENDING | 待处理 |
| PROCESSING | 处理中 |
| COMPLETED | 已完成 |
| CANCELLED | 已取消 |
````

**公共对象定义章节说明**：
- 被多个接口引用的对象 → 抽取到「公共对象定义」章节，各接口通过锚点引用（见 RULES.md 对象定义位置规则）
- 仅被单个接口使用的对象 → 内联定义在该接口中，不放入公共对象定义
- 如果不存在公共对象，则省略「公共对象定义」章节及其目录链接

## 标题层级

| 层级 | Markdown | 用途 |
|------|----------|------|
| 文档标题 | `#` | 文档名称 |
| 一级标题 | `##` | 接口标题 / 公共请求头 / 公共对象定义 / 枚举定义汇总 |
| 二级标题 | `###` | 接口子模块（带三级编号如 1.1.1）/ 公共对象定义下的对象 / 枚举定义汇总下的枚举 |
| 三级标题 | `####` | 接口内的内联对象定义（根对象、嵌套对象） |

## 子模块顺序

按以下顺序输出（跳过不存在的）：

1. 请求信息（始终存在）
2. 请求头（可选）
3. Cookie（可选）
4. 路径参数（可选）
5. 查询参数（可选）
6. 请求体（可选）
7. 响应结构（始终存在）

---

## 响应包装格式

所有响应必须包含外层包装：

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": { }
}
```

分页响应：

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "total": 100,
    "list": [{ }],
    "pageNum": 1,
    "pageSize": 10,
    "pages": 10
  }
}
```

## JSON 示例规范

**强制规则：JSON 示例必须包含对应对象定义中的所有字段，不可省略任何字段。**

- 必填字段：填写合理的示例值
- 可选字段：填写合理的示例值或 `null`
- 枚举字段：使用某个有效的枚举常量值
- 嵌套对象/数组：至少包含一个完整元素（元素内也须包含所有字段）
- 时间字段：使用格式对应的示例值（如 `"2026-01-01 10:00:00"`、`"2026-01-01"`）

````markdown
<!-- ❌ 省略了部分字段 -->
```json
{
  "id": 1,
  "name": "测试"
}
```

<!-- ✅ 包含对象定义的所有字段 -->
```json
{
  "id": 1,
  "name": "测试",
  "status": "ENABLE",
  "statusDesc": "启用",
  "remark": "备注",
  "createTime": "2026-01-01 10:00:00"
}
```
````
