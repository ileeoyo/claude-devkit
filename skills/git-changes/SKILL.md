# Git改动读取助手

读取当前Git项目的所有改动，并根据参数智能执行后续操作。

## 任务参数

$ARGUMENTS

## 参数说明

- **无参数**：仅读取并展示 Git 改动摘要
- **有参数**：读取 Git 改动后，AI 根据参数内容智能分析并执行相应操作

### 常见参数示例

| 参数 | 说明 |
|------|------|
| `生成commit` | 读取改动后，分析内容生成 commit message 并执行 commit |
| `代码审查` | 读取改动后，对代码进行审查并给出建议 |
| `生成变更文档` | 读取改动后，生成本次改动的变更说明文档 |
| `staged` | 仅读取已暂存的改动 |
| `unstaged` | 仅读取未暂存的改动 |
| `<文件路径>` | 仅读取指定文件的改动 |
| `<任意描述>` | AI 智能理解意图并执行相应操作 |

## 执行步骤

### 第一阶段：获取Git改动信息

#### 1. 获取Git状态和改动内容

执行一条命令获取完整改动信息：
- `git status && git diff HEAD` - 查看文件状态及所有改动详情（含已暂存和未暂存）
- 如有未跟踪的新文件需要分析， 需要具体读取其内容

#### 3. 输出改动摘要

整理并输出以下信息：
- 改动涉及的文件列表
- 新增文件
- 修改文件
- 删除文件
- 各文件的改动概要（新增/删除行数）
- 主要改动内容摘要

### 第二阶段：根据参数执行后续操作

**如果无参数**：输出改动摘要后结束，等待用户下一步指示。

**如果有参数**：AI 分析参数内容，智能判断用户意图并执行相应操作：

- 若参数包含"commit"、"提交"等关键词：
  - 分析改动内容，生成符合规范的 commit message
  - 将未暂存的改动添加到暂存区
  - 执行 git commit 命令

- 若参数包含"审查"、"review"等关键词：
  - 对改动代码进行审查
  - 指出潜在问题和改进建议

- 若参数包含"文档"、"changelog"等关键词：
  - 生成本次改动的变更说明文档

- 若参数为文件路径：
  - 仅读取并展示该文件的改动详情

- 其他参数：
  - AI 智能理解用户意图并执行相应操作

## 输出格式

```
## Git改动摘要

**分支**: [当前分支名]
**改动文件数**: [数量]

### 文件列表

| 状态 | 文件路径 | 改动 |
|------|----------|------|
| M/A/D | path/to/file | +X -Y |

### 改动详情

[按文件列出主要改动内容]
```
